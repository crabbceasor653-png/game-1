# 05 - 验收标准

## 启动

- [x] `npm run dev` 可以启动本地开发服务器。
- [x] 打开页面后能看到标题界面。
- [x] 页面没有明显报错阻断运行。

## 玩法

- [x] 可以开始新游戏。
- [x] 键盘可以控制移动。
- [x] 鼠标拖动可以控制移动。
- [x] 触屏拖动可以控制移动。
- [x] 敌人会持续刷新。
- [x] 玩家会自动射击。
- [x] 新开局玩家生命为 7，移动速度为 225，经验吸附半径为 125。
- [x] 前 2 分钟敌人刷新更慢，Runner 和 Brute 不会过早出现。
- [x] 碰撞会掉血。
- [x] 生命归零会进入结算。
- [x] 可以重开。
- [x] 收集 5 个怪兽经验球后自动触发 `星环齐射`。
- [x] `星环齐射`持续约 3 秒，并向 360° 多方向发射子弹。
- [x] 技能激活期间继续充能时，持续时间最多延长到 4.5 秒。

## 成长

- [x] 经验可以累积。
- [x] 达到阈值会进入升级选择。
- [x] 至少有 3 个可选升级。
- [x] 升级会真实影响玩法。
- [x] `相位护盾`、`磁场收束`、`再生电池`能出现在升级池，并真实影响玩法。

## 存储和反馈

- [x] 最高分会保存到本地。
- [x] 静音开关会保存到本地。
- [x] 音效首次交互后可用。

## 兼容性

- [x] 1280x720 桌面尺寸布局正常。
- [x] 390x844 手机尺寸布局正常。
- [x] 窗口缩放后仍可正常游玩。
- [x] 不出现严重遮挡或文字溢出。

## 体验

- [x] 一局至少能完整跑通 3 分钟的生存过程。
- [x] 场景切换清楚。
- [x] 反馈足够让玩家知道自己发生了什么。

## 验收证据

- `npm run smoke` 覆盖真实浏览器启动、标题界面、开始游戏、键盘移动、鼠标拖动、触屏拖动、敌人刷新、自动射击、升级选择、暂停、结算、最高分刷新持久化、桌面尺寸和移动端尺寸。
- `npm run smoke` 必须额外覆盖降低难度数值、五经验自动触发 `星环齐射`、360° 子弹、技能持续/延长、以及三种新增技能升级效果。
- `node --check` 覆盖 `src/main.js`、`scripts/dev-server.mjs`、`scripts/devlog.mjs`、`scripts/browser-smoke.mjs` 的语法完整性。
- 3 分钟生存过程由 `npm run smoke` 中的 `simulateSurvival(180)` 推进真实游戏更新逻辑覆盖。
