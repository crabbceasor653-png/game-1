# 04 - 执行步骤

## Phase 1 - 项目骨架

1. 建立 `index.html`、`styles.css`、`src/main.js`。
2. 建立 `scripts/dev-server.mjs`。
3. 建立 `scripts/devlog.mjs`。
4. 建立 `scripts/browser-smoke.mjs`。
5. 建立 `devlogs/` 和第一天日志。

验收：

- `npm run dev` 能启动本地页面。
- 页面能加载文档和基础 UI。
- `npm run smoke` 能完成浏览器冒烟验证。

## Phase 2 - 战斗核心

1. 让玩家能移动。
2. 让敌人自动刷出并追击。
3. 让玩家自动射击。
4. 让子弹、碰撞、伤害和死亡可工作。

验收：

- 可以无升级地生存并被击败。
- 分数会随着游玩增长。

## Phase 3 - 成长系统

1. 加入经验值和等级。
2. 加入升级选择界面。
3. 加入多条强化路线。
4. 加入掉落物和吸附机制。

验收：

- 一局内会多次升级。
- 每次升级都能做出选择。

## Phase 4 - 界面和音效

1. 做标题界面。
2. 做暂停、继续、结算界面。
3. 做 HUD 和状态条。
4. 做基本音效与静音开关。

验收：

- 所有主界面可见且可操作。

## Phase 5 - 存档与兼容

1. 保存最高分和设置。
2. 处理窗口缩放和移动端适配。
3. 检查性能和布局。

验收：

- 刷新后最高分仍在。
- 桌面和移动端都能完成一局。
