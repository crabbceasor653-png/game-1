# 03 - 设计规范

## 视觉方向

- 深色宇宙感底色。
- 青蓝、玫红、酸橙、琥珀作为强调色。
- 主要信息密度高，但层次清楚。
- 画面以发光线条、粒子和简洁几何体为主。

## 颜色

- 背景：`#071018`
- 面板底：`rgba(10, 18, 30, 0.88)`
- 主文字：`#e9f4ff`
- 次文字：`#91a5bf`
- 青蓝强调：`#31d9ff`
- 玫红强调：`#ff4db8`
- 成功提示：`#9cff57`
- 警告提示：`#ffbb4d`

## 布局

- 主画面必须全屏。
- UI 只做必要覆盖，不做大面积装饰。
- 顶部展示分数、等级、生命、经验。
- 右上角放暂停、重开、静音。
- 中央只在标题、升级、结算、暂停时出现覆盖层。

## 组件规则

- 按钮使用清晰的边框和高对比文字。
- 圆角不超过 8px。
- 不用花哨卡片堆叠。
- 不做冗余的装饰性背景泡泡。
- 文本不能溢出容器。

## 字体

- 使用系统无衬线字体栈。
- 正文默认 16px 左右。
- 标题可以更大，但不要依赖视口宽度缩放。

## 反馈

- 开火、受击、升级、死亡都要有明确视觉反馈。
- 粒子要轻量。
- 画面可闪烁，但不能影响可读性。

## 响应式

- 桌面优先，移动端同样可用。
- 竖屏也要能玩。
- 小屏上先保证可操作，再保证信息完整。
